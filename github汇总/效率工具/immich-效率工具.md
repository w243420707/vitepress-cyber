# 项目名称
Immich

# 作者或组织名
immich-app

# 项目类别
效率工具

# 核心痛点
Google Photos等云相册服务存在隐私问题，且数据不由自己控制，缺乏好的自托管替代方案

# 主要功能
- 高性能自托管照片和视频管理解决方案
- 功能类似Google Photos
- 支持手机端和Web端
- 自动备份、防止重复、选择性相册备份
- 多用户支持、相册和共享相册
- 支持RAW格式、元数据查看（EXIF、地图）
- 通过元数据、物体、人脸、CLIP搜索
- 人脸识别和聚类、回忆功能
- 全球地图、伙伴分享、归档和收藏
- 支持LivePhoto/MotionPhoto、360度图像
- 用户自定义存储结构、公开分享
- 多语言支持（包括简体中文）

# 项目亮点
- 功能非常全面，几乎涵盖Google Photos的所有功能
- 自托管，数据完全由自己控制
- 高性能，处理大量照片视频也很流畅
- 支持多语言，包括简体中文
- 手机端和Web端都有
- 完全免费开源

# 安装或部署教程

## 教程适用对象与环境范围
适用于有Docker使用经验的用户，推荐Linux系统

## 前置条件检查
1. **Docker检查**
   - 步骤目标：确认已安装Docker和Docker Compose
   - 执行位置：本机终端
   - 完整命令：`docker --version` 和 `docker-compose --version`
   - 成功标志：显示Docker和Docker Compose版本号
   - 失败处理：访问 [Docker官网](https://www.docker.com/) 安装

## 主路径：Docker Compose部署
README/仓库信息中未提供完整的一步一步教程，以下为基于项目文档的通用步骤：

### 步骤1：准备部署
- **步骤目标**：准备部署环境
- **执行位置**：本机终端
- **操作说明**：参考官方安装文档准备环境
- **成功标志**：环境准备完成
- **失败处理**：查看官方文档

### 步骤2：下载配置文件
- **步骤目标**：下载Docker Compose配置文件
- **执行位置**：部署目录
- **操作说明**：从官方文档下载docker-compose.yml和.env文件
- **成功标志**：配置文件下载完成
- **失败处理**：检查网络连接

### 步骤3：配置环境变量
- **步骤目标**：编辑.env文件配置环境变量
- **执行位置**：部署目录
- **操作说明**：编辑.env文件，设置必要的配置项
- **成功标志**：配置完成
- **失败处理**：参考官方文档配置

### 步骤4：启动服务
- **步骤目标**：启动Docker Compose服务
- **执行位置**：部署目录
- **完整命令**：
  ```bash
  docker-compose up -d
  ```
- **成功标志**：服务启动成功
- **失败处理**：查看Docker日志排查错误

## 安装成功验证
- **访问地址**：http://YOUR_SERVER_IP:2283
- **预期返回**：Immich Web界面正常显示

## 常见错误排查
1. **现象**：服务启动失败
   - **可能原因**：端口被占用、配置错误
   - **解决步骤**：检查端口占用，查看.env配置

2. **现象**：无法上传照片
   - **可能原因**：存储空间不足、权限问题
   - **解决步骤**：检查存储空间，确认文件权限

3. **现象**：手机端无法连接
   - **可能原因**：网络问题、服务器地址错误
   - **解决步骤**：检查网络连接，确认服务器地址正确

# 项目名称
gogcli

# 作者或组织名
steipete

# 项目类别
开发工具

# 核心痛点
需要在终端中访问Google的各种服务，进行脚本自动化和日常操作，而不需要打开浏览器。

# 主要功能
- 访问Gmail：搜索线程/消息、发送邮件、查看附件、管理标签/草稿/过滤器/委托/假期设置
- 访问Calendar：列出/创建/更新/删除事件、管理邀请、别名、订阅、团队日历等
- 访问Chat：列出/查找/创建空间、列出消息/线程、发送消息和DM、管理表情反应
- 访问Classroom：管理课程、名册、课程作业/材料、提交、公告、主题、邀请、监护人、个人资料
- 访问Drive：列出/搜索/上传/下载文件、替换上传、转换上传、管理权限/评论、组织文件夹
- 访问Contacts：搜索/创建/更新联系人，包括地址、关系、组织/职位元数据、自定义字段
- 访问Tasks：管理任务列表和任务：获取/创建/添加/更新/完成/撤销/删除/清除
- 访问Sheets：读/写/更新电子表格、插入行/列、管理标签和命名范围、格式化/合并/冻结/调整单元格大小
- 访问Forms：创建/更新表单、管理问题、检查响应、管理监视
- 访问Apps Script：创建/获取/绑定项目、检查内容、运行函数
- 访问Docs/Slides：创建/复制/导出文档/幻灯片、编辑文档、导入Markdown、导出文档为Markdown/HTML
- 访问People：个人资料查找和目录搜索帮助
- 访问Keep：列出/获取/搜索/创建/删除笔记和下载附件
- 访问Admin：Workspace Admin用户/组命令，用于常见目录操作
- 访问Groups：列出你所属的组，查看组成员
- 本地时间：为脚本和代理显示快速本地/UTC时间

# 项目亮点
- 多账户支持：同时管理多个Google账户，支持账户别名和每个客户端的OAuth存储桶
- 命令允许列表：限制沙盒/代理运行的顶级命令
- 安全的凭证存储：使用OS密钥环或加密的磁盘密钥环
- 自动刷新令牌：一次认证，无限期使用
- 灵活的认证：OAuth刷新令牌、ADC、直接访问令牌、服务账户、手动/远程流程
- 最少权限认证：--readonly、--drive-scope和--gmail-scope请求更少的范围
- JSON优先输出：便于脚本和自动化

# 安装或部署教程

## 教程适用对象与环境范围
适用于需要在终端中访问Google服务的开发者和用户，支持Linux、macOS和Windows。

## 前置条件检查
1. 系统要求：Linux、macOS或Windows
2. 依赖：Git（从源代码构建时需要）
3. 权限要求：有足够的权限安装软件

## 主路径：安装与启动

### 通过Homebrew安装
1. **安装gogcli**
   - 执行位置：终端
   - 完整命令：`brew install gogcli`
   - 成功标志：安装完成，无错误信息
   - 失败处理：检查Homebrew是否正确安装

### 通过Arch User Repository安装
1. **安装gogcli**
   - 执行位置：终端
   - 完整命令：`yay -S gogcli`
   - 成功标志：安装完成，无错误信息
   - 失败处理：检查yay是否正确安装

### 从源代码构建
1. **克隆仓库**
   - 执行位置：终端
   - 完整命令：`git clone https://github.com/steipete/gogcli.git`
   - 成功标志：仓库克隆成功
   - 失败处理：检查网络连接

2. **构建项目**
   - 执行位置：项目目录
   - 完整命令：`cd gogcli && make`
   - 成功标志：构建完成，无错误信息
   - 失败处理：检查系统环境是否满足要求

3. **运行gogcli**
   - 执行位置：项目目录
   - 完整命令：`./bin/gog --help`
   - 成功标志：显示帮助信息
   - 失败处理：检查构建是否成功

## 配置与使用

### 获取OAuth2凭证
1. **创建Google Cloud项目**
   - 执行位置：浏览器
   - 操作：访问 https://console.cloud.google.com/projectcreate 创建项目
   - 成功标志：项目创建成功
   - 失败处理：检查Google Cloud Console访问权限

2. **启用API**
   - 执行位置：浏览器
   - 操作：启用需要的API，如Gmail API、Calendar API、Drive API等
   - 成功标志：API启用成功
   - 失败处理：检查项目权限

3. **配置OAuth同意屏幕**
   - 执行位置：浏览器
   - 操作：访问 https://console.cloud.google.com/auth/branding 配置OAuth同意屏幕
   - 成功标志：配置完成
   - 失败处理：检查项目设置

4. **创建OAuth客户端**
   - 执行位置：浏览器
   - 操作：访问 https://console.cloud.google.com/auth/clients 创建桌面应用类型的OAuth客户端
   - 成功标志：客户端创建成功，下载JSON文件
   - 失败处理：检查项目配置

### 存储凭证
1. **存储OAuth客户端凭证**
   - 执行位置：终端
   - 完整命令：`gog auth credentials ~/Downloads/client_secret_....json`
   - 成功标志：凭证存储成功
   - 失败处理：检查文件路径是否正确

### 授权账户
1. **授权账户**
   - 执行位置：终端
   - 完整命令：`gog auth add you@gmail.com`
   - 成功标志：浏览器打开OAuth授权页面，授权完成后返回终端
   - 失败处理：检查网络连接，确保浏览器可以打开

2. **无头/远程服务器流程**
   - 执行位置：终端
   - 完整命令：`gog auth add you@gmail.com --services user --manual`
   - 成功标志：终端打印认证URL，在本地浏览器打开并授权，然后将重定向URL粘贴回终端
   - 失败处理：检查网络连接

### 测试认证
1. **测试认证**
   - 执行位置：终端
   - 完整命令：`export GOG_ACCOUNT=you@gmail.com && gog gmail labels list`
   - 成功标志：显示Gmail标签列表
   - 失败处理：检查认证是否成功

## 安装成功验证
- 执行命令：`gog --version`
- 预期返回：显示gogcli版本

## 常见错误排查
1. **认证失败**
   - 现象：执行命令后出现认证错误
   - 可能原因：OAuth凭证过期、权限不足
   - 解决步骤：重新授权账户，确保启用了所需的API

2. **API调用失败**
   - 现象：执行命令后出现API错误
   - 可能原因：API未启用、权限不足
   - 解决步骤：检查是否启用了所需的API，确保授权时请求了正确的范围

3. **依赖安装失败**
   - 现象：从源代码构建时出现依赖错误
   - 可能原因：缺少必要的依赖
   - 解决步骤：检查系统环境，安装所需的依赖

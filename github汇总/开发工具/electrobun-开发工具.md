# electrobun

## 项目名称
electrobun

## 作者或组织名
blackboardsh

## 项目类别
开发工具

## 核心痛点
传统桌面应用开发需要学习多种语言、打包体积大、更新麻烦。

## 主要功能
使用TypeScript构建超快、超小、跨平台桌面应用的完整解决方案。使用Bun作为主进程执行引擎，用Zig编写原生绑定。打包体积约12MB，更新包可小至14KB。提供主进程和Webview进程之间的隔离，支持快速、类型安全的RPC通信。支持macOS、Windows和Linux三大平台。提供模板和完整的开发工作流。

## 项目亮点
- 打包体积约12MB，更新包可小至14KB
- 使用Bun执行引擎和Zig原生绑定
- 主进程和Webview进程隔离
- 快速、类型安全的RPC通信
- 支持macOS、Windows、Linux
- 5分钟开始写代码，10分钟发布应用

## 安装或部署教程

### 教程适用对象与环境范围
本教程适用于想要使用Electrobun开发桌面应用的开发者。

### 前置条件检查

1. **检查Node.js版本**
   - 执行命令：`node -v`
   - 通过标准：显示Node.js版本号

2. **检查Bun是否安装**
   - 执行命令：`bun -v`
   - 通过标准：显示Bun版本号

### 主路径：使用模板快速开始

**步骤1：初始化项目**
- 目标：使用模板创建新项目
- 位置：终端
- 命令：`npx electrobun init`
- 成功标志：项目创建成功
- 失败处理：检查网络连接

**步骤2：进入项目目录**
- 目标：进入项目文件夹
- 位置：终端
- 命令：`cd <项目名称>`
- 成功标志：进入项目目录
- 失败处理：确认项目名称正确

**步骤3：安装依赖**
- 目标：安装项目依赖
- 位置：项目目录
- 命令：`bun install`
- 成功标志：依赖安装完成
- 失败处理：检查Bun是否正确安装

**步骤4：运行开发模式**
- 目标：启动开发服务器
- 位置：项目目录
- 命令：`bun dev`
- 成功标志：应用启动成功
- 失败处理：检查端口是否被占用

### 备选路径：从源码构建

**步骤1：克隆项目**
- 目标：获取源码
- 位置：终端
- 命令：`git clone --recurse-submodules https://github.com/blackboardsh/electrobun.git`
- 成功标志：项目克隆成功
- 失败处理：检查网络连接 [海外网络环境配置 →](/网络环境)

**步骤2：安装依赖**
- 目标：安装项目依赖
- 位置：项目目录
- 命令：`cd electrobun/package && bun install`
- 成功标志：依赖安装完成
- 失败处理：检查Bun版本

**步骤3：运行开发环境**
- 目标：启动开发环境
- 位置：项目目录
- 命令：`bun dev:clean`
- 成功标志：开发环境启动成功
- 失败处理：检查系统依赖

### 安装成功验证
- 应用窗口正常打开
- 可以看到Electrobun默认界面
- 热重载功能正常工作

### 常见错误排查

1. **Bun命令不存在**
   - 可能原因：Bun未安装
   - 解决步骤：运行 `curl -fsSL https://bun.sh/install | bash` 安装Bun

2. **应用无法启动**
   - 可能原因：系统依赖缺失
   - 解决步骤：安装对应平台的开发工具（macOS需要Xcode命令行工具）

3. **打包失败**
   - 可能原因：缺少平台特定工具
   - 解决步骤：安装对应平台的构建工具

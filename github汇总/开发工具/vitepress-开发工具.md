# 核心痛点

- 基于Vite的静态站点生成器，启动和构建速度超快

- 自动生成侧边栏导航
- 完整支持Vue组件，可在Markdown中直接使用
- 一键初始化项目，配置简单
- 支持自定义主题和插件扩展

# 安装或部署教程

## 教程适用对象与环境范围
本教程适用于想要使用VitePress搭建技术文档或博客的开发者，支持Windows、macOS和Linux系统。

## 前置条件检查

### 1. Node.js版本检查
- 步骤目标：确认Node.js版本满足要求
- 执行位置：本机终端
- 完整命令：`node --version`
- 成功标志：显示Node.js 16.0或更高版本
- 失败处理：请安装或升级Node.js至16.0+版本

### 2. npm或pnpm检查
- 步骤目标：确认包管理器可用
- 执行位置：本机终端
- 完整命令：`npm --version` 或 `pnpm --version`
- 成功标志：显示包管理器版本信息
- 失败处理：请安装npm或pnpm

### 3. 网络访问检查
- 步骤目标：确认可以访问npm仓库
- 执行位置：本机终端
- 完整命令：`ping registry.npmjs.org`
- 成功标志：可以正常访问
- 失败处理：请检查网络连接或配置npm镜像源

## 主路径：安装与启动

### 步骤1：创建项目目录
- 步骤目标：创建一个空的项目文件夹
- 执行位置：本机终端
- 完整命令：
  ```
  mkdir my-docs
  cd my-docs
  ```
- 成功标志：进入新创建的目录
- 失败处理：确保有文件系统权限

### 步骤2：初始化VitePress项目
- 步骤目标：使用向导初始化VitePress项目
- 执行位置：项目目录终端
- 完整命令：`npx vitepress init`
- 成功标志：启动初始化向导，按提示选择配置
- 失败处理：尝试使用pnpm：`pnpm dlx vitepress init`

### 步骤3：安装依赖
- 步骤目标：安装项目所需的依赖包
- 执行位置：项目目录终端
- 完整命令：`npm install`
- 成功标志：显示安装完成，无错误
- 失败处理：检查网络连接，或使用国内镜像源

### 步骤4：启动开发服务器
- 步骤目标：启动本地开发服务器预览站点
- 执行位置：项目目录终端
- 完整命令：`npm run docs:dev`
- 成功标志：显示服务器启动成功，给出本地访问地址（通常是http://localhost:5173）
- 失败处理：检查端口是否被占用，或查看错误信息

## 安装成功验证

### 1. 访问本地站点
- 访问方式：浏览器
- 访问地址：http://localhost:5173
- 预期返回：看到VitePress默认页面

### 2. 测试热更新
- 访问方式：编辑docs/index.md文件
- 检查方法：修改文件内容后保存，浏览器应自动刷新显示更新
- 预期返回：页面即时更新

### 3. 构建生产版本
- 访问方式：命令行
- 检查命令：`npm run docs:build`
- 预期返回：构建成功，生成dist目录

## 常见错误排查

### 现象1：Node.js版本过低
- 可能原因：Node.js版本不满足最低要求
- 解决步骤：
  1. 检查当前Node.js版本：node --version
  2. 下载并安装Node.js 16.0或更高版本
  3. 重新打开终端并验证版本

### 现象2：依赖安装失败
- 可能原因：网络问题或npm镜像源访问受限
- 解决步骤：
  1. 检查网络连接
  2. 配置国内npm镜像源：npm config set registry https://registry.npmmirror.com
  3. 重新运行npm install

### 现象3：端口被占用
- 可能原因：5173端口已被其他程序占用
- 解决步骤：
  1. 查找占用端口的程序并关闭
  2. 或修改VitePress配置使用其他端口
  3. 重新启动开发服务器

## 备选部署路径

### 使用pnpm安装
- 适用场景：偏好使用pnpm作为包管理器
- 完整命令：
  ```
  pnpm add -D vitepress
  pnpm vitepress init
  ```

### 手动配置
- 适用场景：想要完全自定义配置
- 步骤：
  1. 手动创建package.json
  2. 安装vitepress依赖
  3. 创建docs目录和配置文件

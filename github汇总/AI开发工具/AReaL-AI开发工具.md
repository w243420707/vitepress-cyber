# 核心痛点
- AI Agent使用后能力无法提升
- 缺少智能体边用边训的训练框架

- 不同Agent框架之间兼容性差

# 主要功能
- 智能体强化学习训练框架
- 为OpenClaw等Agent提供边用边训能力
- 只需一个文件即可自定义多轮rollout工作流程
- 可顺畅整合其他智能体工具框架
- 兼容性强，可兼容任意Agent
- 提供灵活性、可扩展性和兼容性三大亮点

# 安装或部署教程

## 教程适用对象与环境范围
本教程适用于想要使用AReaL训练AI Agent的开发者，支持Windows、Linux和MacOS系统。

## 前置条件检查

### 1. Python版本检查
- 步骤目标：确认Python版本满足要求
- 执行位置：本机终端
- 完整命令：`python --version`
- 成功标志：显示Python 3.8或更高版本
- 失败处理：请安装或升级Python至3.8+版本

### 2. Git检查
- 步骤目标：确认Git已安装
- 执行位置：本机终端
- 完整命令：`git --version`
- 成功标志：显示Git版本信息
- 失败处理：请安装Git

### 3. 网络访问检查
- 步骤目标：确认可以访问GitHub
- 执行位置：本机浏览器
- 检查方法：访问https://github.com/inclusionAI/AReaL
- 成功标志：可以正常访问项目页面
- 失败处理：请配置海外网络环境 [海外网络环境配置 →](/网络环境)

## 常见错误排查

### 现象1：Python版本过低
- 可能原因：Python版本不满足最低要求
- 解决步骤：
  1. 检查当前Python版本：python --version
  2. 下载并安装Python 3.8或更高版本
  3. 重新打开终端并验证版本

### 现象2：依赖安装失败
- 可能原因：网络问题或pip镜像源访问受限
- 解决步骤：
  1. 检查网络连接
  2. 配置国内pip镜像源：pip config set registry https://registry.npmmirror.com
  3. 重新运行pip install

### 现象3：运行失败
- 可能原因：配置问题或缺少依赖
- 解决步骤：
  1. 检查配置是否正确
  2. 确认所有依赖已安装
  3. 查看项目Issue页面是否有类似问题

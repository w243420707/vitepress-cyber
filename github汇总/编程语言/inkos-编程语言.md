# 项目名称
inkos

# 作者或组织名
Narcooo

# 项目类别
编程语言

# 核心痛点
开发者需要在Rust的安全性和Lua的灵活性之间做选择，缺乏一个能结合两者优点的语言。

# 主要功能
- 受Rust和Lua影响的小语言
- 与Lua运行时兼容
- 可编译为Lua源代码
- 结合系统语言特性与易用脚本语言

# 项目亮点
- 提供比Lua更安全、更强大的语言选择
- 保持Lua的灵活性和简洁性
- 为喜欢Rust安全特性但需要Lua脚本便利性的开发者提供折中方案
- 可以利用现有的Lua生态系统

# 安装或部署教程

## 教程适用对象与环境范围
适用于对编程语言感兴趣的开发者，支持Rust和Lua环境。

## 前置条件检查
1. Rust环境：已安装Rust和Cargo
2. Lua环境：已安装Lua（如需运行编译结果）

## 主路径：安装编译器

### 步骤1：克隆项目
步骤目标：获取项目代码
执行位置：终端
完整命令或操作：
```bash
git clone https://github.com/Narcooo/inkos.git
cd inkos
```
成功标志：克隆成功，进入项目目录
失败处理：检查网络连接，确认Git安装

### 步骤2：编译项目
步骤目标：编译inkos编译器
执行位置：项目目录
完整命令或操作：
```bash
cargo build --release
```
成功标志：编译成功完成
失败处理：检查Rust版本，更新依赖

## 安装成功验证
1. 编译器能正常运行
2. 能编译示例inkos代码为Lua
3. 编译后的Lua代码能正常运行

## 常见错误排查

1. **现象 -> 编译失败**
   可能原因 -> Rust版本不兼容或依赖问题
   解决步骤 -> 更新Rust到最新版本，查看项目依赖要求

2. **现象 -> 生成的Lua代码运行错误**
   可能原因 -> inkos语法问题或兼容性问题
   解决步骤 -> 查看inkos语法文档，确认代码符合要求

3. **现象 -> 功能不完整**
   可能原因 -> 项目还在开发中
   解决步骤 -> 查看项目状态，了解当前支持的功能

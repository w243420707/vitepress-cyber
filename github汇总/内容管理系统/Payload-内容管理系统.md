# 项目名称
Payload

# 作者或组织名
Payload CMS Team

# 项目类别
内容管理系统

# 核心痛点
传统CMS部署复杂、前后端分离、vendor lock-in、功能扩展困难

# 主要功能
- Next.js原生，可直接安装在app文件夹中
- 既是应用框架又是无头CMS
- 一键部署到Vercel或Cloudflare
- 前后端可在同一app文件夹
- 内置认证、版本控制、草稿、本地化
- 块级布局构建器
- 可自定义的React管理面板
- Lexical富文本编辑器
- 条件字段逻辑
- 精细的访问控制
- 文档和字段级钩子

# 项目亮点
- 首个Next.js原生CMS
- 完全开源免费，无vendor lock-in
- 前后端一体，开发体验流畅
- 一键部署，无需复杂配置
- 管理面板和后端100%可扩展
- 内置丰富功能，开箱即用

# 安装或部署教程

## 教程适用对象与环境范围
适用于想要使用Payload构建网站、电商、博客等应用的开发者，需要Node.js环境。

## 前置条件检查
1. 系统要求：Windows 10及以上、macOS 10.15及以上、主流Linux发行版
2. Node.js版本：最新LTS版本
3. 包管理器：pnpm、npm或yarn
4. 网络要求：能够访问npm仓库

## 主路径：快速开始（推荐新手）

### 步骤1：创建新项目
**步骤目标**：创建一个新的Payload项目
**执行位置**：命令行终端
**完整命令或操作**：运行 `pnpx create-payload-app@latest`
**成功标志**：项目创建成功
**失败处理**：检查Node.js和pnpm是否正确安装

### 步骤2：选择模板
**步骤目标**：选择合适的项目模板
**执行位置**：命令行终端
**完整命令或操作**：根据提示选择模板，推荐使用website模板
**成功标志**：模板选择完成
**失败处理**：查看官方文档了解模板详情

### 步骤3：启动开发服务器
**步骤目标**：启动Payload开发服务器
**执行位置**：项目目录
**完整命令或操作**：进入项目目录，运行 `pnpm dev`
**成功标志**：开发服务器启动成功
**失败处理**：检查端口是否被占用，依赖是否正确安装

### 步骤4：访问管理面板
**步骤目标**：访问Payload管理面板
**执行位置**：浏览器
**完整命令或操作**：访问 http://localhost:3000/admin
**成功标志**：管理面板正常显示
**失败处理**：检查开发服务器是否正常运行

## 安装成功验证
访问管理面板，能够正常登录和使用各项功能。

## 常见错误排查
1. **现象**：依赖安装失败
   - **可能原因**：网络问题或包管理器问题
   - **解决步骤**：检查网络连接，尝试更换npm镜像源

2. **现象**：开发服务器启动失败
   - **可能原因**：端口被占用或配置问题
   - **解决步骤**：检查3000端口是否被占用，查看错误日志

3. **现象**：管理面板无法访问
   - **可能原因**：开发服务器未正常运行
   - **解决步骤**：确认开发服务器正在运行，检查防火墙设置

## 备选部署路径：一键部署
可直接通过Vercel或Cloudflare一键部署，无需本地安装。

# 项目名称
Lux

# 作者或组织名
iawia002和其他贡献者

# 项目类别
视频工具

# 核心痛点
需要为不同平台寻找不同的视频下载工具，下载过程中可能遇到速度慢、断点续传困难等问题。

# 主要功能
- 支持多个视频平台的视频、图片和音频下载
- 多线程下载，提高下载速度
- 播放列表下载功能
- 断点续传能力
- 代理设置支持
- Cookie支持，用于访问需要登录的内容
- 多种安装方式，适合不同操作系统

# 项目亮点
- 使用Go语言构建，性能优异
- 支持众多主流视频平台
- 提供多种安装方式，方便用户使用
- 命令行界面简洁易用
- 开源免费，采用MIT许可证

# 安装或部署教程

## 教程适用对象与环境范围
适用于需要下载网络视频、图片和音频的用户，支持Windows、macOS和Linux操作系统。

## 前置条件检查
1. 系统要求：Windows、macOS或Linux
2. 依赖：FFmpeg（用于视频合并，非必须）
3. 网络要求：能够访问目标视频平台

## 主路径：安装与启动

### Windows系统
1. **使用Scoop安装**
   - 执行位置：Windows命令行
   - 完整命令：`scoop install lux`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查Scoop是否已安装，若未安装，先安装Scoop

2. **使用Chocolatey安装**
   - 执行位置：Windows命令行
   - 完整命令：`choco install lux`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查Chocolatey是否已安装，若未安装，先安装Chocolatey

### macOS系统
1. **使用Homebrew安装**
   - 执行位置：macOS终端
   - 完整命令：`brew install lux`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查Homebrew是否已安装，若未安装，先安装Homebrew

### Linux系统
1. **使用Cask安装**
   - 执行位置：Linux终端
   - 完整命令：`cask install github.com/iawia002/lux`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查Cask是否已安装，若未安装，先安装Cask

2. **Arch Linux用户**
   - 执行位置：Arch Linux终端
   - 完整命令：通过AUR安装lux
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查AUR helper是否已安装

3. **Void Linux用户**
   - 执行位置：Void Linux终端
   - 完整命令：`xbps-install -S lux`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查包管理器是否正常

### 所有系统通用
1. **使用go install安装**
   - 执行位置：终端
   - 完整命令：`go install github.com/iawia002/lux@latest`
   - 成功标志：命令执行完成，无错误信息
   - 失败处理：检查Go是否已安装，版本是否满足要求

2. **从Releases页面下载二进制文件**
   - 执行位置：浏览器
   - 操作：访问GitHub Releases页面，下载对应系统的二进制文件
   - 成功标志：下载完成，文件可执行
   - 失败处理：检查网络连接，确认下载链接正确

## 安装成功验证
- 执行命令：`lux -v`
- 预期返回：显示lux的版本信息

## 常见错误排查
1. **下载失败**
   - 现象：执行下载命令后出现错误
   - 可能原因：网络问题、目标平台限制、Cookie过期
   - 解决步骤：检查网络连接，尝试使用代理，更新Cookie

2. **视频合并失败**
   - 现象：下载完成后无法播放视频
   - 可能原因：缺少FFmpeg
   - 解决步骤：安装FFmpeg，重新下载

3. **播放列表下载不完整**
   - 现象：只下载了部分视频
   - 可能原因：网络中断、平台限制
   - 解决步骤：使用断点续传功能，重新执行下载命令

## 备选部署路径
直接从GitHub Releases页面下载对应系统的二进制文件，无需安装依赖。

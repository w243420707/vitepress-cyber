# 项目名称
pi-mono

# 作者或组织名
Mario Zechner（badlogic）

# 项目类别
AI工具

# 核心痛点
开发AI智能体时需要对接多个LLM提供商，管理复杂；缺少统一的智能体运行时、UI组件等基础设施，开发效率低。

# 主要功能
构建AI智能体和管理LLM部署的工具包，以Monorepo形式包含统一多提供商LLM API、智能体运行时、终端UI等七个包。

# 项目亮点
1. 统一的多提供商LLM API，一套代码可调用20多个大模型
2. 智能体运行时支持工具调用和状态管理
3. 提供交互式编程智能体命令行工具
4. 包含终端UI库和Web组件，快速构建界面
5. 提供管理vLLM部署的CLI工具
6. Monorepo结构，七个实用包打包在一起

# 安装或部署教程

## 教程适用对象与环境范围
适用于AI智能体开发者、LLM应用开发者，需要Node.js和npm环境。

## 前置条件检查
1. 检查Node.js和npm是否安装：
   - 执行位置：本机终端
   - 完整命令：`node -v && npm -v`
   - 成功标志：显示Node.js和npm版本号
   - 失败处理：安装Node.js和npm

## 主路径：安装与启动

### 步骤1：克隆仓库
- 步骤目标：获取项目代码
- 执行位置：项目目录
- 完整命令：`git clone https://github.com/badlogic/pi-mono.git && cd pi-mono`
- 成功标志：仓库克隆成功，进入项目目录
- 失败处理：检查网络连接，确认GitHub地址正确

### 步骤2：安装依赖
- 步骤目标：安装项目所有依赖
- 执行位置：项目根目录
- 完整命令：`npm install`
- 成功标志：依赖安装完成
- 失败处理：检查网络连接，尝试使用淘宝镜像

### 步骤3：构建项目
- 步骤目标：构建所有包
- 执行位置：项目根目录
- 完整命令：`npm run build`
- 成功标志：构建完成，无错误
- 失败处理：检查Node.js版本是否符合要求

### 步骤4：代码检查
- 步骤目标：执行lint、格式化和类型检查
- 执行位置：项目根目录
- 完整命令：`npm run check`
- 成功标志：检查通过，无错误
- 失败处理：根据错误提示修复问题（注意npm run check需要先执行npm run build）

## 安装成功验证
1. 访问方式：终端命令
2. 检查命令：`./pi-test.sh`（从源码运行pi，必须在仓库根目录执行）
3. 预期返回：pi正常运行
4. 页面特征（如适用）：无

## 常见错误排查
1. 现象：npm run check失败
   - 可能原因：未先执行npm run build
   - 解决步骤：先运行npm run build，再运行npm run check

2. 现象：依赖安装失败
   - 可能原因：网络问题或npm镜像问题
   - 解决步骤：尝试使用淘宝npm镜像或检查网络连接

3. 现象：构建失败
   - 可能原因：Node.js版本不兼容
   - 解决步骤：检查Node.js版本，升级到推荐版本

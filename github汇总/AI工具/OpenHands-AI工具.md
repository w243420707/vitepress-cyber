# 安装或部署教程

## 教程适用对象与环境范围
适用于想要使用OpenHands进行AI辅助开发的开发者，支持Windows、macOS、Linux。

## 前置条件检查
1. 系统要求：Windows 10及以上、macOS 10.15及以上、主流Linux发行版
2. Python版本：Python 3.10+或3.11+
3. 网络要求：能够访问LLM提供商
4. API密钥：如使用云端LLM，需要对应API密钥

## 主路径：快速开始（推荐新手）

### 步骤1：选择使用方式
**步骤目标**：决定使用OpenHands的方式
**执行位置**：浏览器
**完整命令或操作**：
- 云端：访问 https://cloud.all-hands.dev
- 本地UI：下载并安装OpenHands
**成功标志**：选择好使用方式
**失败处理**：查看README了解各方式区别

### 步骤2：配置LLM
**步骤目标**：配置要使用的LLM
**执行位置**：OpenHands界面
**完整命令或操作**：在设置中添加LLM配置，如OpenAI的API Key
**成功标志**：LLM配置保存成功
**失败处理**：检查API Key是否正确，网络连接是否正常

### 步骤3：设置工作目录
**步骤目标**：设置OpenHands的工作目录
**执行位置**：OpenHands界面
**完整命令或操作**：选择或创建工作目录
**成功标志**：工作目录设置完成
**失败处理**：检查目录权限

### 步骤4：开始使用
**步骤目标**：让OpenHands帮助开发
**执行位置**：OpenHands界面
**完整命令或操作**：在聊天框中输入需求，让OpenHands执行
**成功标志**：OpenHands开始工作
**失败处理**：检查LLM配置是否正确

## 安装成功验证
OpenHands能够正常响应并执行任务。

## 常见错误排查
1. **现象**：LLM连接失败
   - **可能原因**：API Key错误或网络问题
   - **解决步骤**：检查API Key和网络连接，或参考 [海外网络环境配置 →](/网络环境)

2. **现象**：无法执行文件操作
   - **可能原因**：权限问题或工作目录设置
   - **解决步骤**：检查工作目录权限和设置

3. **现象**：本地UI无法启动
   - **可能原因**：安装不完整
   - **解决步骤**：重新安装或使用云端版本

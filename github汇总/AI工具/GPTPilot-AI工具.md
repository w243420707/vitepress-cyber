# 安装或部署教程

## 教程适用对象与环境范围
适用于想要使用GPT Pilot进行AI辅助开发的开发者，支持Windows、macOS、Linux。

## 前置条件检查
1. 系统要求：Windows 10及以上、macOS 10.15及以上、主流Linux发行版
2. Python版本：Python 3.10+
3. 网络要求：能够访问LLM提供商
4. API密钥：如使用云端LLM，需要对应API密钥

## 主路径：快速开始（推荐新手）

### 步骤1：克隆仓库
**步骤目标**：获取GPT Pilot代码
**执行位置**：命令行终端
**完整命令或操作**：运行 `git clone https://github.com/Pythagora-io/gpt-pilot.git && cd gpt-pilot`
**成功标志**：代码克隆完成
**失败处理**：检查网络连接，或参考 [海外网络环境配置 →](/网络环境)

### 步骤2：创建虚拟环境
**步骤目标**：创建Python虚拟环境
**执行位置**：命令行终端
**完整命令或操作**：运行 `python -m venv venv`
**成功标志**：虚拟环境创建成功
**失败处理**：检查Python是否正确安装

### 步骤3：激活虚拟环境
**步骤目标**：激活Python虚拟环境
**执行位置**：命令行终端
**完整命令或操作**：
- Windows：运行 `venv\Scripts\activate`
- macOS/Linux：运行 `source venv/bin/activate`
**成功标志**：命令行提示符前出现(venv)
**失败处理**：检查虚拟环境是否正确创建

### 步骤4：安装依赖
**步骤目标**：安装GPT Pilot的依赖
**执行位置**：命令行终端
**完整命令或操作**：运行 `pip install -r requirements.txt`
**成功标志**：依赖安装完成
**失败处理**：检查网络连接，尝试更换pip镜像源

### 步骤5：配置API密钥
**步骤目标**：配置LLM API密钥
**执行位置**：命令行终端或代码编辑器
**完整命令或操作**：复制.env.example为.env，填入API密钥
**成功标志**：配置文件保存成功
**失败处理**：检查API Key是否正确

### 步骤6：运行GPT Pilot
**步骤目标**：启动GPT Pilot
**执行位置**：命令行终端
**完整命令或操作**：运行 `python main.py`
**成功标志**：GPT Pilot启动成功
**失败处理**：检查依赖是否正确安装，配置是否正确

## 安装成功验证
GPT Pilot正常启动并等待用户输入。

## 常见错误排查
1. **现象**：LLM连接失败
   - **可能原因**：API Key错误或网络问题
   - **解决步骤**：检查API Key和网络连接，或参考 [海外网络环境配置 →](/网络环境)

2. **现象**：依赖安装失败
   - **可能原因**：网络问题或Python版本不兼容
   - **解决步骤**：检查Python版本，使用国内pip镜像源

3. **现象**：虚拟环境激活失败
   - **可能原因**：操作系统不同
   - **解决步骤**：使用对应操作系统的激活命令
